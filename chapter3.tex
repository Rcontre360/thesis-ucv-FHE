\chapter{Propuesta de Trabajo Especial de Grado}

El esquema de cifrado CKKS es una de las soluciones más eficientes al momento de trabajar con operaciones con números de punto flotante \cite{CKKS17}. Sin embargo, son pocas las implementaciones del mismo que aprovechan la GPU. Actualmente, las bibliotecas más prominentes que implementan el esquema CKKS usan exclusivamente la CPU \cite{SEAL, Halevi14, PALISADE}; aun así, recientemente se han desarrollado bibliotecas que implementan este esquema de cifrado usando la GPU. Entre estas bibliotecas está HEonGPU \cite{HEonGPU}, la cual implementa varios esquemas de cifrado homomórfico, incluyendo CKKS con bootstrapping.

Los modelos de aprendizaje automático basados en redes neuronales que utilizan cifrado homomórfico usualmente usan los esquemas CKKS en la CPU \cite{SEAL} o TFHE en la GPU \cite{Concrete}. Usualmente, se aprovecha la privacidad de estas operaciones durante la inferencia. En el entrenamiento también es posible usar cifrado homomórfico; sin embargo, es menos costoso usar modelos ya entrenados y luego aplicar el cifrado durante la inferencia. Los modelos de redes neuronales pueden llegar a tener millones o cientos de millones de parámetros, esto hace costoso el cómputo durante la inferencia y poco viable si se usa solo la CPU. Además, usar cifrado homomórfico ya implica un costo mayor para cada operación realizada. Dependiendo del esquema utilizado, una operación de suma o multiplicación es más costosa que aplicarla sin usar un esquema de cifrado, y este costo es proporcional al nivel de seguridad del cifrado \cite{Acar18}.

Para implementar redes neuronales que realicen operaciones con datos cifrados existen distintas herramientas. La mayoría de ellas son bibliotecas que nos permiten usar varios esquemas de cifrado (CKKS, BFV, TFHE), que usan la CPU \cite{SEAL, Halevi14, PALISADE}. Existen algunas otras que permiten usar la GPU, como Concrete \cite{Concrete}; sin embargo, este esquema fue diseñado trabajar sobre booleanos y no reales. Existen algunas implementaciones recientes del esquema CKKS sobre la GPU, estas implementan las operaciones básicas como suma, multiplicación, rotación, etc. \cite{Turkoglu22, HEonGPU}. Sin embargo, no existen herramientas que faciliten el uso de estas bibliotecas en modelos de aprendizaje automático. Por este motivo se propone la creación de una biblioteca que permita usar el esquema de CKKS, aprovechando el rendimiento de la GPU, sobre modelos de aprendizaje automático.

\section{Objetivo general}

Desarrollar una biblioteca de código abierto en Python para el cómputo de operaciones tensoriales, utilizando el esquema de cifrado homomórfico CKKS de forma que se aproveche la unidad de procesamiento gráfico (GPU).

\section{Objetivos específicos}

\begin{itemize}
    \item Diseñar una interfaz en Python que permita interactuar con las bibliotecas FIDESlib y OpenFHE, y usar las operaciones básicas del esquema de cifrado CKKS.
    \item Implementar las clases y módulos que permitan realizar operaciones con tensores y matrices, abstrayendo la complejidad del esquema CKKS del usuario.
    \item Crear casos de uso donde se ejemplifique cómo usar la biblioteca para aprovechar el cifrado homomórfico durante la inferencia en modelos de aprendizaje automático.
    \item Comparar el rendimiento de capas de redes neuronales implementadas con capas que usan otras bibliotecas de cifrado homomórfico.
\end{itemize}

\section{Tecnologías necesarias}

El desarrollo de la biblioteca propuesta es un sistema de múltiples capas de abstracción, cuyo objetivo es cerrar la brecha entre los entornos de programación de alto nivel utilizados en aprendizaje automático y la ejecución de operaciones criptográficas complejas aceleradas por hardware. Para lograr esto, se emplearán librerias que permitan usar el esquema CKKS en el GPU y que nos permitan usar el esquema desde el lenguaje Python.

\begin{itemize}
    \item FIDESlib: Desarrollada en C++ y CUDA, esta biblioteca proporciona una implementación optimizada del esquema CKKS sobre el GPU \cite{FIDESlib}. 
    
    \item OpenFHE-Python y Pybind11: Para usar las primitivas matemáticas de FIDESlib en Python, se utilizará la interfaz oficial OpenFHE-Python \cite{OpenFHEPython}. Esta herramienta actúa como interface, empleando Pybind11 para crear \textit{bindings} que exponen los objetos complejos de C++ como objetos nativos de Python. La incorporación de Pybind11 es indispensable para servir como mecanismo formal de traducción de tipos y gestión de memoria entre ambos lenguajes, permitiendo además la personalización de \textit{bindings} en caso de requerir optimizaciones específicas para la manipulación de tensores o características exclusivas de FIDESlib.
\end{itemize}

Finalmente, la biblioteca propuesta se desarrollara en Python, abstrayendo la complejidad criptográfica para el usuario mediante estructuras de datos como tensores cifrados. Las operaciones sobre estos tensores invocarán los \textit{bindings} de OpenFHE-Python, los cuales delegarán la validación criptográfica a OpenFHE y el cálculo intensivo a la GPU a través de FIDESlib. Se emplearán bibliotecas adicionales del ecosistema de Python, como Matplotlib, para la visualización y análisis de los resultados de rendimiento.

\section{Modelos a implementar y comparativas}

Una vez finalizado el desarrollo de la biblioteca, el enfoque se centrará en la implementación de diferentes arquitecturas de redes neuronales para validar su funcionalidad. Específicamente, se construirán redes neuronales totalmente conectadas con un número variable de capas y neuronas, diseñadas tanto para tareas de regresión como de clasificación. Además, se implementará una red neuronal convolucional orientada a tareas de clasificación. Durante estas pruebas, también se evaluará el impacto de cambiar las funciones de activación dentro de estos modelos cifrados.

Para validar la efectividad y eficiencia de la solución propuesta, se llevarán a cabo dos tipos de comparativas principales:

\begin{enumerate}
    \item \textbf{Resultados} Se compararán los resultados de inferencia obtenida por los modelos usando cifrado homomórfico, frente a los mismos modelos operando sobre datos sin cifrado. Esto permitirá medir la pérdida de precisión inducida por las aproximaciones polinómicas y la gestión del ruido del esquema CKKS.
    \item \textbf{Rendimiento:} Se contrastarán los tiempos de ejecución y uso de memoria de la biblioteca propuesta contra otras herramientas disponibles en Python que permiten usar cifrado homomórfico, como TenSEAL \cite{benaissa2021tenseal} (la cual implementa CKKS) y Concrete ML \cite{ConcreteML} (basada en el esquema de cifrado TFHE). Es importante destacar que TenSEAL usa exclusivamente el CPU mientras que Concrete ML usa el GPU pero usa un esquema de cifrado distinto a CKKS.
\end{enumerate}
