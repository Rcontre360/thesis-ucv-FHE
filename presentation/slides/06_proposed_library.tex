\section{Propuesta de Trabajo}
\SectionFrame{Propuesta de Trabajo}

\begin{frame}{Objetivos y Stack Tecnológico}
    \begin{itemize}
        \item \textbf{Objetivo:} Desarrollar una biblioteca Python para tensores cifrados (CKKS) acelerada por GPU (CUDA).
        \item \textbf{Abstracción:} Ocultar la complejidad criptográfica (Relinearización, Reescalado) tras una interfaz tipo NumPy/PyTorch.
        \item \textbf{Stack Tecnológico:}
        \begin{itemize}
            \item \textbf{FIDESlib:} Backend optimizado en CUDA/C++ para el esquema CKKS.
            \item \textbf{OpenFHE-Python:} Interface oficial de OpenFHE.
            \item \textbf{Pybind11:} Gestión de memoria y bindings entre C++ y Python.
        \end{itemize}
    \end{itemize}
\end{frame}

\begin{frame}{Casos de Uso y Benchmarking}
    Para validar la funcionalidad y el rendimiento, se implementarán y evaluarán los siguientes modelos:
    \begin{itemize}
        \item \textbf{Modelos DNN:} Redes totalmente conectadas para tareas de regresión y clasificación.
        \item \textbf{Modelos CNN:} Red neuronal convolucional para clasificación de imágenes.
    \end{itemize}
    
    \begin{block}{Métricas de Comparación (vs. TenSEAL CPU)}
        \begin{enumerate}
            \item \textbf{Rendimiento Temporal:} Tiempo total de inferencia (Latencia).
            \item \textbf{Uso de Recursos:} Consumo de VRAM vs. Memoria RAM convencional.
            \item \textbf{Precisión Numérica:} Impacto de las aproximaciones polinómicas y el error de redondeo de CKKS frente al modelo en texto plano.
        \end{enumerate}
    \end{block}
\end{frame}
