\section{El Esquema CKKS}
\SectionFrame{El Esquema CKKS}

\begin{frame}{¿Por qué el Esquema CKKS?}
    Propuesto en 2017, CKKS es un esquema diseñado para el cómputo sobre números reales y complejos.
    \vspace{0.3cm}
    \begin{itemize}
        \setlength{\itemsep}{12pt}
        \item \textbf{Aritmética Aproximada:} CKKS trata el ruido inherente como un ``error de redondeo controlado''.
        \item \textbf{Mayor precisión:} Permite procesar datos decimales de forma nativa sin necesidad de cuantizar los datos cifrados.
        \item \textbf{Ideal para Redes Neuronales:} Las RNAs son inherentemente robustas a pequeños errores de precisión, lo que hace que CKKS sea la elección natural para este dominio.
    \end{itemize}
\end{frame}

\begin{frame}{Operaciones fundamentales en CKKS}
    \begin{itemize}
        \setlength{\itemsep}{12pt}
        \item \textbf{Adición:} Operación lineal directa; el consumo de presupuesto de ruido es despreciable.
        \item \textbf{Multiplicación:} Operación cuadrática que consume niveles del esquema.
        \begin{itemize}
            \item \textbf{Relinearización:} Reduce el tamaño del texto cifrado tras la multiplicación.
            \item \textbf{Reescalado:} Gestiona el factor de escala y elimina bits de ruido residual.
        \end{itemize}
        \item \textbf{Rotación:} Permite permutar los elementos dentro de los vectores cifrados, permitiendo realizar operaciones de álgebra lineal de forma eficiente.
    \end{itemize}
\end{frame}

\begin{frame}{Cifrado por niveles}
    Permite realizar cómputos de profundidad fija predefiniendo los parámetros del esquema.
    \vspace{0.2cm}
    \begin{itemize}
        \item \textbf{Cadena de Módulos:} $Q_L \rightarrow Q_{L-1} \rightarrow \dots \rightarrow Q_0$.
    \end{itemize}

\begin{frame}{Bootstrapping}
    El \textit{bootstrapping} es el proceso que permite realizar cómputos de profundidad infinita.
    \vspace{0.3cm}
    \begin{itemize}
        \setlength{\itemsep}{12pt}
        \item \textbf{Refresco de ruido:} Toma un texto cifrado con mucho ruido y genera uno nuevo con ruido mínimo.
        \item \textbf{Descifrado homomórfico:} Se evalúa homomórficamente la propia función de descifrado del esquema.
    \end{itemize}
