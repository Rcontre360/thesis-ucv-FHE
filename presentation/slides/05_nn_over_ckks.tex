\section{Inferencia Privada con CKKS}

\begin{frame}{Inferencia Privada}
    En el contexto de FHE, el cifrado se aplica generalmente al proceso de inferencia ya que los pesos del modelo suelen ser públicos, mientras que los datos del usuario son sensibles.
    \begin{itemize}
        \item \textbf{Flujo:} Cliente cifra entrada $\rightarrow$ Servidor evalúa homomórficamente $\rightarrow$ Cliente descifra.
        \item \textbf{Ventaja:} El servidor realiza el cómputo intensivo sin conocer los datos ni el resultado.
        \item \textbf{Limitación del Entrenamiento:} Actualizar pesos requiere retropropagación y cálculo de gradientes, lo cual es prohibitivo en FHE debido al crecimiento del ruido y la profundidad del circuito.
    \end{itemize}

\end{frame}

\begin{frame}{Multiplicación Matriz-Vector y Diagonalización}
    \begin{itemize}
        \item Operación lineal costosa en el dominio homomórfico.
        \item Técnica de Diagonalización (Halevi y Shoup) para minimizar rotaciones.
        \item Optimiza el uso de la estructura SIMD.
    \end{itemize}

\begin{frame}{Aproximación de Funciones de Activación}
    \begin{itemize}
        \item El esquema CKKS solo soporta sumas y multiplicaciones polinomiales.
        \item \textbf{Desafío:} Funciones no lineales (ReLU, Sigmoid) no pueden evaluarse directamente.
        \item \textbf{Soluciones de Aproximación:}
        \begin{itemize}
            \item \textbf{Series de Taylor:} Útiles para funciones suaves (Sigmoid, Tanh).
            \[ \text{Sigmoid}(x) \approx \frac{1}{2} + \frac{1}{4}x - \frac{1}{48}x^3 + \dots \]
            \item \textbf{Polinomios de Bajo Grado:} Reducen la profundidad del circuito.
            \[ \text{ReLU}(x) \approx x^2 \text{ (CryptoNets)} \quad \text{o} \quad ax^2 + bx + c \]
            \item \textbf{Mínimos Cuadrados:} Ajuste óptimo en un intervalo definido.
        \end{itemize}
    \end{itemize}
\end{frame}

\begin{frame}{Convoluciones Homomórficas e Im2Col}
    \begin{itemize}
        \item Transformación del tensor de entrada y filtros mediante \textit{Im2Col}.
        \item La convolución se convierte en una multiplicación matriz-vector optimizada.
        \item Permite aprovechar el paralelismo SIMD del esquema.
    \end{itemize}
